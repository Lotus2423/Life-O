# Contributing
# Contributing to TwinForge AI

Thank you for your interest in contributing to TwinForge AI.

We welcome contributions that improve the project while maintaining high engineering standards.

---

## Development Workflow

1. Fork the repository
2. Create a new feature branch
3. Write clean and documented code
4. Test your changes
5. Submit a Pull Request

---

## Branch Naming

feature/dashboard

feature/backend

feature/ai

feature/digital-twin

fix/api

docs/readme

---

## Commit Message Convention

feat: Add new feature

fix: Fix bug

docs: Update documentation

refactor: Improve code

test: Add tests

style: Code formatting

chore: Maintenance task

---

## Coding Standards

Python

- Follow PEP8
- Use Type Hints
- Add Docstrings

React

- Functional Components
- TypeScript
- Reusable Components

General

- Write meaningful variable names
- Keep functions small and focused
- Avoid duplicate code

---

## Pull Request Checklist

- Code builds successfully
- Tests pass
- Documentation updated
- No unnecessary files included
- Code reviewed before submission

---

## Reporting Issues

When reporting a bug, include:

- Steps to reproduce
- Expected behavior
- Actual behavior
- Logs or screenshots
- Environment details

---

## Community Guidelines

- Be respectful
- Help others
- Keep discussions technical
- Focus on constructive feedback

---

## Vision

TwinForge AI is an educational and engineering-focused open-source project that aims to demonstrate modern Industry 4.0 concepts through practical implementation.

Every contribution should improve code quality, documentation, or user experience.
Please open issues and pull requests.
